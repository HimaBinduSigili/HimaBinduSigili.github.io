Robot Model:

Unicycle (rotating and translating underactuated robot) with configuration q = (x, y, ÆŸ), (6-dimensional state space with 2 dofs in configuration space) non-holonomic motion model with motion constraint, I have implemented the collision detection myself without using any collision detection package as my obstacles are mostly circular or elliptical (trees as mentioned earlier). For 3D implementation I have used Klampt.

I used python as my programming language for this project. Starting with a point robot (assuming holonomic motions) and implemented RRT for holonomic motions just to kick start the project and later implemented algorithm for non-holonomic motions. For better performance I have used the bi-directional RRT as it is a goal biased search which yields efficient results. I have not considered special cases like the pits (if there any in the farm) as I used planar terrain for simplicity, this non planar terrain case can be considered as polygonal obstacles in 2D implementation and can be achieved in future works.
